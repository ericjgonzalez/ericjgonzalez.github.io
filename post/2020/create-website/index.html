Introduction

This post is my documentation for setting up a website using the R package Blogdown, the static site generator Hugo, and Git version control. The following instructions were sourced from the Blogdown book and the Hugo quick start guide.

Getting Started

GitHub

  • Create two new Github repositories for a User Pages site.
    • The first repository will contain the project directory for the website.
    • The second repository will contain the publishing source files and will be named <user>.github.io. Replace <user> with your user account name.
  • Identify a Hugo theme and fork the chosen theme’s GitHub repository.
    • For example, this website is powered by Hugo-Academic.

RStudio

  • Create a new project in RStudio.
    • Select Website using blogdown.
    • Directory name will be identical to the first GitHub repository.
    • Hugo theme will be gcushen/hugo-academic.
    • Enable package management for your project with renv.
    # install.packages("renv")
renv::init()
renv::install("blogdown")
blogdown::install_hugo()
    • Create a .gitignore file.
    file.create(".gitignore")
file <- file(".gitignore")
# add content to file
writeLines(c(".Rproj.user", ".Rhistory", ".RData", ".Ruserdata", "/public"),
           file)
close(file)
    • Delete themes/hugo-academic to add your forked GitHub theme.
    # in terminal
git rm themes/hugo-academic

Git

  • Create new Git repository.
    • Initialize repository for your .Rproj directory.
    # in terminal
cd /d 'path-to-Rproj-directory'
git init
git add .
git commit -m "initial commit"
git remote add origin 'path-to-GitHub-project-repo'
git push origin master
    • Add Hugo theme as Git submodule.
    # in terminal
git submodule add 'path-to-forked-GitHub-theme-repo' themes/hugo-academic
git push origin master
    • Update theme submodule when necessary.
    # in terminal
cd /d 'path-to-Rproj-directory'
# determine current theme version
"themes/hugo-academic/data/academic.toml"
# update the theme submodule
git submodule update --remote --merge
cd themes/hugo-academic
# you need to consecutively apply any breaking changes from each release
# version upgrading to = vX.X.X
git checkout <VERSION>

Build Website

  • Create new website
    • Serve website in RStudio.
    # creates /public directory
blogdown::serve_site()
    • Edit config.toml baseurl with "https://<user>.github.io/".
    • Add .nojekyll file to /public directory.
    # in terminal
cd /d 'path-to-public-directory'
touch .nojekyll
    • Initialize Git repository for your /public directory.
    # in terminal
cd /d 'path-to-public-directory'
git init
git add .
git commit -m "initial commit"
git remote add origin 'path-to-user.github.io-repo'
git push origin master
